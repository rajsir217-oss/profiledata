Design rationale

Hypothesis: The dashboard's primary job is to make users discover new people. Stats and inbox-management are secondary; surface as much fresh content as possible.

  • Match of the day hero — anchors the visit. Rotates daily, picks highest L3V3L score from new arrivals.
  • Netflix-style horizontal rows — each row is a curated lens: New this week, Saved search results (one row per saved search!), Favorited you, Mutual interest, etc.
  • Compact match cards with L3V3L score badge + online dot + "New" badge. Hover/click to view profile.
  • Saved searches become first-class content — instead of a stat card showing "5 saved searches", each one becomes a row of fresh results.
  • Inbox/stats relegated to a small utility strip at the bottom — they're a 1-click escape, not the main event.
  • Banners → notification drawer (bell, top-right). Same as Mockups A & B.

Trade-offs: Heavier API load (more profiles fetched per visit). Visually busy if user has many saved searches (could become 10+ rows of scroll). Less optimal for "I just want to reply to messages" users — they have to scroll to bottom.
