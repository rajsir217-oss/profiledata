Design rationale

Hypothesis: Users open the dashboard to catch up — "what changed since I was last here?" Optimize for chronological awareness and inline action.

  • Timeline-first (LinkedIn / GitHub / Twitter pattern). Everything is an event with timestamp + actor + actions inline.
  • "23 things happened since 2 days ago" — anchors the user's mental model. No more guessing what's new.
  • Inline actions on every event card — Reply / Approve / View. No modal opens, no page navigation needed for routine tasks.
  • Filter tabs let power users zoom into one event type without losing context.
  • Left sidebar = navigation by type with live counts (replaces accordion sections).
  • Right rail = passive stats + nudges + poll, all collapsed.
  • L3V3L matches appear as event cards in-flow (highlighted), not as a separate section.

Trade-offs: Heavier scroll-load. Less suitable for users who only check in monthly (timeline becomes overwhelming). Search must move to top bar (still 1 click away).
